# Task brief: OpenVideo hybrid AI editor product direction

**Issue:** #40
**Branch:** `docs/40-openvideo-hybrid-ai-direction`

## Outcome

Record the approved OpenVideo direction: a hybrid AI video editor built on a local-first editing workflow, with optional local models or explicitly selected external AI services.

## Scope

- Define the local editing workflow as the primary system of record.
- Define assisted editing and optional generation capability groups.
- Establish that an external AI connection is user-selected and operation-specific.
- Separate planned AI features from the currently released local editor.
- Establish public-copy, accessibility, SEO, and verification requirements for a later implementation PR.

## Non-goals

- Runtime website copy changes.
- Desktop AI feature, model, provider, account, analytics, or cloud-storage implementation.
- Claims that AI features have already shipped.

## Acceptance checks

- The specification contains no vague placeholders or contradictory current-state claims.
- The specification explicitly requires accurate availability language before future AI capabilities ship.
- The documentation-only diff is scoped to the product-direction specification and task brief.
